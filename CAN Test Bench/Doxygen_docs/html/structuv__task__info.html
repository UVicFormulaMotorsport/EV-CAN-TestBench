Detailed Description

This struct is designed to hold neccessary information about an RTOS task that will be managed by uvfr_state_engine.

task_id is an internal ID assigned to every task and is used by uvfr_utils. task_name is the name of the task. This must be less than 15 characters, and ended with a null terminator.

task_function is a pointer to the function we want the task to run task_priority is the priority of the task instances is the number of legal instances

task_state is an internal representation of what the task is up to. This is needed because the RTOS task may not actually exist. active_states dictates when the states are active

task_handle is an osThreadId created by FreeRTOS when it initializes the task. next exists because this shit needs to exist in a linked list, and my ass is too lazy to create a generic struct with it

cmd_data stores the command we really want this task to execute
