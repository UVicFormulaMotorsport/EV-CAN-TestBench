\doxysubsection{Detailed Description}
This struct is designed to hold neccessary information about an R\+T\+OS task that will be managed by uvfr\+\_\+state\+\_\+engine. 

{\ttfamily task\+\_\+id} is an internal ID assigned to every task and is used by uvfr\+\_\+utils. {\ttfamily task\+\_\+name} is the name of the task. This must be less than 15 characters, and ended with a null terminator.

{\ttfamily task\+\_\+function} is a pointer to the function we want the task to run {\ttfamily task\+\_\+priority} is the priority of the task {\ttfamily instances} is the number of legal instances

{\ttfamily task\+\_\+state} is an internal representation of what the task is up to. This is needed because the R\+T\+OS task may not actually exist. {\ttfamily active\+\_\+states} dictates when the states are active

{\ttfamily task\+\_\+handle} is an os\+Thread\+Id created by Free\+R\+T\+OS when it initializes the task. {\ttfamily next} exists because this shit needs to exist in a linked list, and my ass is too lazy to create a generic struct with it

{\ttfamily cmd\+\_\+data} stores the command we really want this task to execute
